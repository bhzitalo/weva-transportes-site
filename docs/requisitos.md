# 📋 Requisitos do Sistema – Weva Transportes

## 1. Introdução

Este documento descreve os requisitos funcionais e não funcionais do sistema a ser desenvolvido para a empresa Weva Transportes.

O objetivo é definir claramente o comportamento esperado do sistema e suas restrições.

## 2. Requisitos Funcionais

Os requisitos funcionais descrevem as funcionalidades que o sistema deve oferecer.

### RF01 – Exibição de Informações Institucionais
O sistema deve apresentar informações sobre a empresa, incluindo missão, visão, valores e história.

### RF02 – Listagem de Serviços
O sistema deve exibir todos os serviços oferecidos pela empresa.

### RF03 – Exibição da Frota
O sistema deve apresentar os veículos disponíveis (vans, ônibus, micro-ônibus e carros).

### RF04 – Página de Contato
O sistema deve disponibilizar uma página com informações de contato da empresa.

### RF05 – Formulário de Contato
O sistema deve permitir que o usuário envie mensagens por meio de um formulário.

### RF06 – Integração com WhatsApp
O sistema deve permitir que o usuário entre em contato diretamente via WhatsApp.

### RF07 – Solicitação de Orçamento
O sistema deve permitir que o usuário solicite um orçamento de transporte.

### RF08 – Divulgação de Viagens
O sistema deve exibir viagens disponíveis ou serviços em destaque.


## 3. Requisitos Não Funcionais

Os requisitos não funcionais definem as características de qualidade do sistema.

### RNF01 – Responsividade
O sistema deve ser responsivo e funcionar corretamente em dispositivos móveis, tablets e desktops.

### RNF02 – Desempenho
O tempo de carregamento das páginas deve ser inferior a 3 segundos em condições normais.

### RNF03 – Usabilidade
A interface deve ser simples, intuitiva e de fácil navegação.

### RNF04 – Compatibilidade
O sistema deve ser compatível com os principais navegadores (Google Chrome, Microsoft Edge, Firefox).

### RNF05 – Segurança
O sistema deve garantir a proteção básica dos dados enviados pelos usuários.

## 4. Requisitos de Interface

### Interface do Usuário
- Layout moderno e limpo
- Uso de cores institucionais (azul escuro, branco e verde)
- Navegação clara com menu visível

### Estrutura de Páginas
- Página Inicial
- Página de Serviços
- Página de Frota
- Página Sobre
- Página de Contato

## 5. Requisitos de Dados

O sistema deve manipular os seguintes dados:

- Nome do usuário
- E-mail
- Telefone
- Mensagem
- Tipo de serviço solicitado

## 6. Considerações Finais

Os requisitos definidos neste documento servirão como base para o desenvolvimento do sistema, garantindo que todas as funcionalidades necessárias sejam implementadas com qualidade e eficiência.
